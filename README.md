# MHW Radar

《怪物猎人：世界》游戏内悬浮窗，实时显示怪物血量、位置、动作信息，仅用作一些特定情况下的出招研究。

## 功能

- 任务计时
- 怪物名称、血量、距离、相对角度
- 怪物动作 ID 及中文招式名
- 黑龙下压值（劫火计数器）
- 招式变化检测与闪烁提示
- 可开关的悬浮窗 UI，透明度调节

## 使用

1. 启动游戏 `MonsterHunterWorld.exe`
2. 运行 `mhw-radar.exe`
3. `Ctrl+Shift+U` 隐藏/显示悬浮窗
4. 点击齿轮图标打开设置面板，开关各项显示

## 构建

```powershell
cargo build --release
```

目标平台：`x86_64-pc-windows-msvc`，仅 Windows。

## 数据

怪物中文名称和招式映射表维护在 `src/data/` 下的 JSON 文件中，编译时嵌入二进制，无需额外配置文件。

## 技术说明

使用 Rust 编写，分为三个核心模块：

- **[memory.rs](src/memory.rs)** — 底层内存 I/O，封装 Windows API 实现进程内存读取、指针链解析
- **[radar.rs](src/radar.rs)** — 数据读取循环，20FPS 独立线程运行，管理游戏进程连接、怪物/玩家数据采集、血量/动作/计时解析，通过共享缓存与 UI 线程通信
- **[main.rs](src/main.rs)** — 基于 `egui`/`eframe` 的悬浮窗渲染入口，全局热键监听，绘制覆盖式 UI

### 关键依赖

| 库                     | 用途                               |
| ---------------------- | ---------------------------------- |
| `eframe` / `egui`      | 即时模式 GUI，实现游戏内透明悬浮窗 |
| `sysinfo`              | 进程扫描与 PID 获取                |
| `process-memory`       | Windows 跨进程内存读写             |
| `winapi`               | 全局热键注册、窗口操作             |
| `serde` / `serde_json` | 编译时 JSON 数据解析               |

### 架构特点

- 数据采集与 UI 渲染分离，后台线程独占内存 I/O，主线程仅轮询共享缓存
- 招式名称查询使用 `HashMap`，编译时 `include_str!` 嵌入 JSON，启动时一次解析，运行时零 I/O
- 怪物动作变化自动检测，变招时悬浮窗闪烁提示
- 覆盖式窗口不捕获鼠标/键盘焦点，不影响游戏操作
